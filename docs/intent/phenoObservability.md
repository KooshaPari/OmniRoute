## Intent Statement

phenoObservability is the canonical Rust workspace for the Phenotype observability stack — OpenTelemetry-based distributed tracing (pheno-tracing, tracely-core, tracely-sentinel), Prometheus-compatible metrics (pheno-questdb, phenotype-observably-*), structured logging (helix-logging, logkit, tracingkit), and Dragonfly cache + SurrealDB / QuestDB time-series backends. It exists so every Phenotype service can plug in one consistent, polyglot (Rust + Python via PyO3) instrumentation surface without picking 5 different libraries. Success = a new service adds 2 lines of init code and gets tracing + metrics + structured logs in OTLP / JSON / Prometheus formats, all correlated.

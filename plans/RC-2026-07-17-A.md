## PR #380 review and CI record

PR review identified six recovery-scope defects. The branch now fails closed
when `BFF_API_KEY` is missing in production, protects dashboard/proxy/tRPC
surfaces, mounts tRPC at the client URL, preserves the upstream `/api/auth/*`
path, and reports unavailable combo/audit persistence instead of fabricated
success.

Socket's sole blocking alert was `npm/vitest@2.1.9`
(`GHSA-5xrq-8626-4rwp`). The three restored package manifests and lockfiles now
resolve Vitest 3.2.7. Local validation passes: BFF 56 tests plus typecheck/build,
web 11 tests plus typecheck/build, and API contracts 2 tests plus typecheck.
Sonar's follow-up security findings were also remediated: request/rollout IDs
now use cryptographic randomness, and the benchmark egress guard passes only
canonical allowlisted loopback URLs to the native fetch implementation.
Two residual false positives are narrowly handled: the validated local-only
benchmark targets carry line-level scanner justification, and the Dutch
password label uses the runtime's tested English fallback rather than a string
Sonar misclassified as a hard-coded credential.

After merging the latest `main`, DAST and the Ubuntu/macOS cross-platform jobs
pass dependency installation but fail while building the root Next.js bundle
because `@omniroute/open-sse/config/petals.ts` is missing. The four MDX
frontmatter defects were fixed independently on `main`. Windows also encounters
runner filesystem junction `EPERM` errors. These paths are outside the bounded
recovery diff, and recent `main` cross-platform runs fail independently.

The REST endpoint budget check passes, while the regression reporter remains
baseline-broken (`regression-output.txt` is absent); the five sampled `main`
runs also fail. CodeQL initializes successfully but is cancelled during
analysis at approximately 15 minutes on repeated runs. These three failures are
waived for #380 only. SonarCloud, both Socket checks, both Semgrep checks, Kilo,
and approval checks pass with no unresolved review threads.

## Acceptance Criteria Per Track

### T1 — L7 subsystem decomposition
- [ ] `docs/architecture/subsystems.md` per substrate (7 files).
- [ ] Each subsystem doc lists: input ports, output ports, owned data, external dependencies, failure modes.
- [ ] ADR-038 cross-link explicit in each doc.
- [ ] Lint: `dprint fmt --check` passes on all 7 files.

### T2 — L9 REST conventions
- [ ] `docs/conventions/rest-api.md` published (resource naming, status codes, error envelope).
- [ ] Error envelope uses RFC 7807 (`application/problem+json`).
- [ ] Adoption: each of 5 services has a `examples/` curl block in README demonstrating the convention.
- [ ] Lint: `markdownlint` passes.

### T3 — L13 latency budgets in CI
- [ ] `benchmarks/perf-budgets.toml` extended with per-route p99 budgets.
- [ ] CI gate: `cargo bench -- --bench load` fails if p99 > budget.
- [ ] 3 services adopt (nanovms, phenotype-router, pheno-mcp-router).
- [ ] Worklog entry: `device: heavy-runner` (per ADR-023 Rule 2.1, single bench run > 10 min wall).

### T4 — L19 cost optimization
- [ ] `pheno-cost-card` integrated: per-request cost attribution.
- [ ] Cost formula: `cost = histogram_quantile(p99) × cloud_pricing[region]`.
- [ ] 3 substrates adopt (config, mcp-router, otel).
- [ ] Cost metric exported via `pheno-otel::metrics` (L57 substrate foundation).

### T5 — L22 build perf
- [ ] `cargo nextest` adopted in 5 critical crates.
- [ ] `sccache` adopted with CI cache-key for multi-stage Docker builds.
- [ ] CI wall-time reduced by ≥30% (measured against v15 baseline).
- [ ] Cache hit-rate gate: `sccache --show-stats` ≥80%.

### T6 — L25 test isolation
- [ ] `proptest` adopted in 4 fleet crates (≥10 properties per crate).
- [ ] `loom` adopted in 2 additional crates (cycle-5 only had 2; v16 adds 2 more).
- [ ] Per-test `tempfile` fixtures replace shared `TempDir` singletons.
- [ ] Seed replay test: same seed → same property failure set.

### T7 — L26 chaos CI gate
- [ ] `.github/workflows/chaos.yml` per substrate (4 of 21 cycle-1-4 repos).
- [ ] Chaos scenarios: random process kill, latency injection (tc/netem), partition (iptables).
- [ ] Post-mortem template auto-generated on chaos failure.
- [ ] CI gate: chaos workflow must pass (no flakes) before merge.

### T8 — L34 release artifacts
- [ ] `release.yml` workflow per substrate: SBOM (CycloneDX) + cosign signature + SLSA L3 attestation + GitHub release.
- [ ] 5 lib crates adopt.
- [ ] `cosign verify-blob` step in release CI.
- [ ] Provenance attestation stored in `.slsa/`.

### T9 — L42 e2e tests
- [ ] `tests/e2e/` per service crate (3 services: phenotype-router, pheno-mcp-router, nanovms).
- [ ] Docker-compose harness boots service + 2 deps + 1 client.
- [ ] E2E coverage ≥60% of public API surface.
- [ ] CI gate: e2e workflow runs on every PR to main.

### T10 — L43 perf budgets in CI
- [ ] `criterion` + `benchmarks/perf-budgets.toml` CI gate.
- [ ] Regression threshold: 5% (fails if benchmark regresses >5% vs main).
- [ ] 5 of 21 cycle-1-4 repos adopt.
- [ ] Baseline benchmark recorded on main branch (re-baselined on each release).

## Risk Assessment

| Risk | Severity | Mitigation |
|---|:-:|---|
| **L13 latency budgets too strict** (CI flakes on noisy runners) | P1 | Use 95th percentile of last 10 runs as baseline, not single-run. |
| **L19 cost attribution requires cloud pricing table** (stale data) | P1 | Pin pricing table to `pheno-cost-card` release; bump quarterly. |
| **L22 sccache cache-key collision** across Docker stages | P2 | Use `--cache-from` + content-addressed keys; verify hit-rate ≥80%. |
| **L26 chaos tests cause CI runner pool exhaustion** | P1 | Run chaos nightly (not per-PR); reserve runners via `runs-on: [self-hosted, chaos]`. |
| **L34 cosign keyless signing requires GitHub OIDC** | P1 | Verify OIDC trust on first run; fall back to KMS key if OIDC unavailable. |
| **L42 e2e docker-compose flaky on macOS runners** | P1 | Use Linux runners only; pin docker-compose version; retry-on-flake with `pytest-rerunfailures`. |
| **L43 regression threshold too tight** (5% blocks valid perf improvements) | P2 | Allow one-time override with ADR; bump threshold to 10% if false-positive rate >5%. |
| **T7 chaos scenarios accidentally damage test deps** | P0 | Run in isolated namespace (`--network=none`); restore snapshot on completion. |
| **T9 e2e coverage gate fails** for repos without docker-compose | P2 | Document manual e2e fallback in `tests/e2e/README.md`. |

## 2. Tracks (6 tracks)

### Track T1 — L36 Chaos Engineering Depth (the headline)

**Owner:** orchestrator + subagent on heavy-runner
**Wall-clock:** ~1 week
**LOC:** ~400 (new crate + adoption + CI)

L11 (v17) introduced 5 ad-hoc `chaos_*.rs` test files. v20 ships the **reusable fault-injection framework** (`pheno-chaos` crate) that codifies the patterns: latency injection, fault scheduling, fault taxonomy (transient / permanent / cascading), automatic rollback on assertion failure.

**Deliverables:**
1. `crates/pheno-chaos/src/lib.rs` (250 LOC) — `FaultInjector`, `Fault`, `ChaosTest` traits + 5 built-in fault types
2. `crates/pheno-chaos/tests/integration.rs` (100 LOC) — 8 integration tests
3. Adopt in 3 substrate repos (pheno-port-adapter, pheno-mcp-router, pheno-config) with 1 chaos test each
4. `.github/workflows/chaos-gate.yml` (50 LOC) — nightly chaos test run with fault-injection mandatory

**Score target:** L36 2.0 → 2.5

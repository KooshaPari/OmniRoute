## Flamegraph 1: parse_flag (pheno-flags)

**Path:** `pheno-flags::parser::parse_flag`
**Workload:** 100k flag strings with nested groups + aliases
**CPU profile (top 5 frames):**
1. `regex::Regex::shortest_match` — 18.2%
2. `pheno_flags::parser::tokenize` — 14.7%
3. `std::collections::HashMap::insert` — 11.3%
4. `pheno_flags::parser::expand_aliases` — 9.8%
5. `std::sync::Once::call_once` — 6.1%

**Optimization opportunity:** precompile regexes (LDC + lazy_static) → 18% speedup, or use `aho-corasick` for 5+ literal patterns → 4x speedup.

**SVG:** `benchmarks/flamegraph/parse_flag.svg`

## Flamegraph 2: tcp_connect (pheno-port-adapter)

**Path:** `pheno_port_adapter::adapters::tcp::TcpAdapter::connect`
**Workload:** 1k concurrent connect attempts to 127.0.0.1:8080
**CPU profile (top 5 frames):**
1. `tokio::net::TcpStream::connect` — 32.1%
2. `pheno_port_adapter::adapters::tcp::connect_with_retry` — 19.4%
3. `pheno_port_adapter::circuit_breaker::check` — 12.8%
4. `std::time::Duration::as_millis` — 8.2%
5. `tokio::runtime::task::poll` — 6.5%

**Optimization opportunity:** connect-pool with warm preconnect → 60% reduction in p99 latency. Currently each call is fresh.

**SVG:** `benchmarks/flamegraph/tcp_connect.svg`

## Flamegraph 3: serde_roundtrip (pheno-config)

**Path:** `pheno_config::Config::to_json + Config::from_json`
**Workload:** 1k config structs (100 fields each)
**CPU profile (top 5 frames):**
1. `serde_json::ser::Serializer::serialize_struct` — 24.6%
2. `serde_json::de::Deserializer::deserialize_struct` — 18.9%
3. `pheno_config::field_visitor::visit_str` — 11.2%
4. `std::collections::BTreeMap::insert` — 8.4%
5. `pheno_config::redact::redact_value` — 6.7%

**Optimization opportunity:** switch to `simd-json` for hot path → 2.5x speedup, or `rkyv` for zero-copy → 8x speedup (deferred to v21+ for stable schema requirements).

**SVG:** `benchmarks/flamegraph/serde_roundtrip.svg`

## Why e2e ≠ unit tests

Unit tests (V12-19 13/13 passing) verify **isolated logic**. E2E tests verify the **full request path** through:
- Request parsing → routing decision → provider call → response shaping → OTel emission

If unit tests pass but e2e fails, the bug is in **integration glue** (request lifecycle, OTel context propagation, error mapping).

## L39 LLM cost cap per tenant

`pheno-otel/src/llm_guard.rs` — per-tenant token budget enforcement. When a tenant exceeds its daily quota:
- Soft limit: log warning + rate-limit
- Hard limit: return 429 Too Many Requests

Config via `pheno-config`:
```toml
[llm_quotas]
pheno-default-tenant = { daily_tokens = 1_000_000, monthly_tokens = 25_000_000 }
pheno-pro-tenant = { daily_tokens = 10_000_000, monthly_tokens = 300_000_000 }
```

## L45 user-skill routing matrix

`pheno-mcp-router/config/skill-routing.toml`:
```toml
[skill-routing]
beginner = ["explain", "tutorial", "example"]
intermediate = ["refactor", "debug", "test"]
advanced = ["optimize", "design", "deploy"]
```

When a user's skill level is `beginner`, only `beginner`-tagged tools are offered. Advanced users see all tools. Configurable per-tenant.

v15 T6 added `.github/workflows/chaos.yml` — runs `cargo test --tests -- --test-threads=1 chaos_` on push+PR for `pheno-port-adapter`. **No gate** — failures are reported but the PR is not blocked.

## This change (cycle 6)

Promote the chaos test job to a **gating** check: PR cannot merge if any `chaos_*` test fails.

### Gate logic

```yaml
- name: chaos gate
  run: |
    cargo test --release --tests -- --test-threads=1 chaos_ --nocapture
    if [ $? -ne 0 ]; then
      echo "::error::chaos test failed — PR cannot merge"
      exit 1
    fi
```

### Allowed bypass paths

1. **Maintainer override** — `pheno-port-adapter` maintainer can mark a chaos test as `#[ignore]` with a justification comment
2. **Flake retry** — single retry for tests in the `flaky::*` namespace
3. **Documented exception** — add entry to `chaos-allowlist.yaml` with link to issue explaining why the test is suppressed

### Bypass audit log

Every bypass (override / flake / exception) is logged to `chaos-bypass-audit.jsonl` with PR number, test name, reason, and actor. Audited weekly by the worklog-schema circle.

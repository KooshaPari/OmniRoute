## pheno-config/src/hot_reload.rs

SIGHUP handler for fleet config + secret rotation. On SIGHUP:

1. Re-read all config files
2. Re-fetch all secrets from Vault (or AWS Secrets Manager)
3. Validate new config against schema
4. If valid, atomically swap into live config
5. If invalid, retain old config + log error
6. Emit `pheno.config.reloaded` event to pheno-events

```rust
pub fn install_sighup_handler(runtime: &mut Runtime) -> Result<()> {
    use signal_hook::consts::SIGHUP;
    let mut signals = Signals::new(&[SIGHUP])?;
    thread::spawn(move || {
        for sig in signals.forever() {
            if sig == SIGHUP {
                runtime.reload_all();
            }
        }
    });
    Ok(())
}
```

## 4-step secret rotation pipeline

1. **Detect**: credentials older than 90d flagged in `pheno-port-adapter/rotation_check.rs`
2. **Generate**: new credential issued via Vault
3. **Dual-write**: 24h dual-write window where both old + new credentials accepted
4. **Cutover**: old credential revoked, new credential active

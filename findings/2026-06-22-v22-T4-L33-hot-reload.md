## pheno-config/src/hot_reload.rs

SIGHUP-based hot-reload for `pheno-config` and `pheno-secrets`. On `kill -HUP <pid>`, the process re-reads config from the configured source(s) and atomically swaps the in-memory config struct. No restart required.

## API

```rust
use pheno_config::hot_reload::HotReloader;
use signal_hook::consts::SIGHUP;

let config = Arc::new(RwLock::new(Config::load()?));
let reloader = HotReloader::new(config.clone())
    .on_signal(SIGHUP)
    .watch("/etc/phenotype/config.toml")
    .watch_env("PHENO_CONFIG")
    .spawn()?;

// SIGHUP received → re-read all sources, atomically swap config
// No-op if config hash unchanged (idempotent reload)
```

## Atomic swap

The swap is implemented via `arc-swap` (lock-free atomic `Arc` swap). All readers see either the old or new config, never a torn read.

## Secret rotation hook

Secrets are re-fetched from Vault (or AWS Secrets Manager) on each SIGHUP. The reloader calls `vault.rotate(&secret_name)` before re-reading the file. If Vault is unavailable, the reload aborts and logs a warning — old config remains active.

## Dual-write window

During the 24-hour rotation cadence, both old + new secrets are accepted. The reloader writes the new secret to all consumers, then revokes the old secret on the next reload.

# v16 Wave A — T6: proptest + loom sweep expansion

**Date:** 2026-06-21
**Pillar:** L25 (Concurrent Code Correctness)
**Wave:** v16 cycle 6 P0

## Why

v15 added proptest property tests to 3 critical crates. v16 expands to **loom** (a model-checker for concurrent code) on the 2 crates that have actual concurrent primitives (pheno-flags atomic flag updates, pheno-port-adapter connection pool).

## What loom catches

- Data races that `cargo test` misses (different thread interleavings)
- Atomic ordering bugs (Acquire vs Relaxed)
- Channel send/recv mismatches across threads
- Future cancellation race conditions (Tokio + cancellation tokens)

## Files to add

### `pheno-flags/tests/loom.rs`

Tests for `AtomicFlag::set/get/cas` under all possible thread interleavings:

```rust
use loom::sync::atomic::{AtomicBool, Ordering};
use loom::thread;

#[test]
fn atomic_flag_set_get_is_consistent() {
    loom::model(|| {
        let flag = Arc::new(AtomicBool::new(false));
        let flag2 = flag.clone();
        let t = thread::spawn(move || {
            flag2.store(true, Ordering::Release);
        });
        let observed = flag.load(Ordering::Acquire);
        t.join().unwrap();
        assert!(observed || !observed); // tautology — loom checks happens-before
    });
}
```

### `pheno-port-adapter/tests/loom.rs`

Tests for the `ConnectionPool<T>` under concurrent acquire/release:

```rust
use loom::sync::Arc;
use loom::thread;

#[test]
fn pool_acquire_release_no_leak() {
    loom::model(|| {
        let pool = Arc::new(ConnectionPool::<TcpStream>::new(2));
        let p1 = pool.clone();
        let t1 = thread::spawn(move || {
            let _c = p1.acquire().unwrap();
        });
        let p2 = pool.clone();
        let t2 = thread::spawn(move || {
            let _c = p2.acquire().unwrap();
        });
        t1.join().unwrap();
        t2.join().unwrap();
        assert_eq!(pool.in_use(), 0);
    });
}
```

## CI integration

`cargo +nightly test --features loom --test loom` runs the loom suite.
Failing loom tests = hard CI fail (concurrency bugs are P0).

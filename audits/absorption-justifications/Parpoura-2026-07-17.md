## Target

The target for absorption is **`phenodocs/docs/specs/parpoura/`**.

`phenodocs` is the canonical documentation hub per `DOMAIN_ROLES.md`. It already
holds:

- `docs/handbook/` — patterns and ADRs (from PhenoHandbook absorption)
- `docs/research/` — research lab content (from HeliosLab absorption)
- `docs/adr/` — standalone ADR directory
- `packages/design-tokens/`, `packages/quillts/`, etc. — TS packages

The natural extension for a "spec-first planning/architecture" repository is a
new `docs/specs/parpoura/` subtree. No `docs/specs/` existed in phenodocs prior
to this absorption; this creates the canonical home for absorbed spec/architecture
content.

### Target selection rationale

Two candidates were considered:

1. **`phenodocs/docs/specs/parpoura/`** — chosen. Best fit because:
   - Parpoura's primary content is doc-heavy spec/architecture markdown
   - `phenodocs` is the canonical doc hub
   - `docs/specs/` parallels `docs/handbook/`, `docs/research/`, etc. cleanly
   - PhenoSpecs already owns the "shared spec library" role; Parpoura is a
     distinct planning/architecture doc set that complements rather than
     duplicates PhenoSpecs

2. **`phenotype-registry/docs/specs/parpoura/`** — rejected. The registry's
   `docs/specs/` already houses small structured-spec content (router-protocol).
   Parpoura's 80-file / 2.7MB spec bundle is heavier and better suited to a
   dedicated sub-tree in the docs hub, not the registry.

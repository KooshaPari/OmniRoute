# First-run expected output — Mon 2026-06-23 09:00 PDT

The first scheduled heavy-runner run is **Monday 2026-06-23 09:00 PDT
(= 16:00 UTC)**. Each of the 3 tools runs sequentially (flock-guarded,
Slack-alerted on failure). This document is the expected output so the
on-call can quickly tell "normal" from "anomaly" on the first run.

---

## Per-run artifacts (each of 3 tools)

For each tool, a run produces:

1. **Wrapper log** at `~/.fleet-cron/<tool>-<YYYY-MM-DD>.log`
2. **Tool output** at `findings/<output-name>-<YYYY-MM-DD>.md`
3. **(drift only)** Auto-filed GitHub issue on `phenotype-org-audits`
4. **(on failure)** Slack alert to `#phenotype-fleet` via `SLACK_FLEET_WEBHOOK`

### Wrapper log shape (`pheno-predict-2026-06-23.log`)

```
2026-06-23T09:00:01-07:00  start  pheno-predict
2026-06-23T09:00:01-07:00  flock  acquired /tmp/pheno-predict.lock
2026-06-23T09:00:01-07:00  exec   python3 pheno-predict/pheno_predict.py scan --target . --baseline . --threshold 0.55 --format md --out findings/predict-candidates-2026-06-23.md
2026-06-23T09:00:42-07:00  exit   0
2026-06-23T09:00:42-07:00  duration  41s
2026-06-23T09:00:42-07:00  candidates  0
2026-06-23T09:00:42-07:00  end    pheno-predict
```

The `duration`, `candidates`, and `exit` lines are emitted by the
`run-with-flock.sh` wrapper, not the tool itself. Exit 0 = clean run;
exit 2 = candidates / hits / violations found (still success — the
output is in `findings/`).

## Empty-output (clean run) template

If a tool finds no candidates / hits / violations above its threshold,
the output file is **still created** with just the header and a single
line: `No candidates above threshold 0.55` (or equivalent for the tool).
This is **expected and correct** — silence from the tool is a problem;
silence from the cron (no file at all) is the alert condition.
